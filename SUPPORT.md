# Support

## Getting Help

For usage questions or troubleshooting, open a GitHub Discussion or Issue and include:

- What you are building
- The command you ran
- Relevant logs or error output
- Your `pack version` and `docker version`

## Bug Reports

Open an Issue with clear reproduction steps and expected versus actual behavior.

## Security Issues

Do not open a public issue for vulnerabilities.
Use the process in SECURITY.md.

## Feature Requests

Open an Issue describing the use case, expected behavior, and why it is valuable.

## Response Expectations

Maintainers review requests on a best-effort basis.
Critical security matters are prioritized.
